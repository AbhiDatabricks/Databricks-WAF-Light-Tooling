3. Data Flow — System Tables → waf_cache → Consumers
What this shows: Data does NOT flow directly from System Tables to the dashboard. The WAF Reload Job reads System Tables, evaluates WAF controls, and writes results into pre-built waf_cache Delta tables. Consumers (Dashboard, Genie, App) read only from waf_cache. The _hist tables preserve every run. Views always show the latest run.

4. Genie AI Space — How It Works
What this shows: Genie is a native Databricks AI assistant — no custom training, no embeddings, no vector search. It works by being pre-configured with table access + natural-language instructions at creation time via the serialized_space payload. Genie translates user questions into SQL, runs them against waf_cache, and returns results in plain English.

6. waf_cache Schema (145 Delta Tables)
What this shows: All 145 tables written to {catalog}.waf_cache. Each pillar has the same 3-table pattern: controls, principal percentages, total percentage. Every table has a _hist counterpart for history. The _run_log table auto-increments run_id on each reload.

8. Score Calculation Logic (per pillar)
What this shows: How WAF scores are computed inside waf_reload.py. Each WAF control is a SQL query against System Tables. The result is compared to a threshold. Individual pass/fail rolls up to principal percentage, which rolls up to total pillar score.
